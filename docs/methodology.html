1. Overview

The US Construction Materials Intelligence Report is a programmatically generated HTML dashboard covering the supply chain for cement and supplementary cementitious materials (SCMs) in the United States. It is designed to update automatically when underlying data is refreshed — every chart, statistic, and map marker is generated at build time from structured data sources rather than maintained manually.

The report covers five material categories:

  • Portland cement — the dominant binding agent in US concrete
  • Ground granulated blast furnace slag (GGBFS) — a co-product of iron making that substitutes 30–70% of cement
  • Fly ash — a coal combustion by-product used as a pozzolanic cement replacement
  • Natural pozzolans — volcanic materials (pumice, perlite, diatomite, calcined clays) used as SCMs
  • Silica fume — an ultra-fine by-product of silicon/ferrosilicon production

The supply chain for these materials intersects iron & steel production (slag), electricity generation (fly ash), mining (natural pozzolans), and maritime/rail logistics (imports). This report captures all five layers in a single deliverable.

The output is a single, self-contained HTML file (~56 KB). No server, database, or build framework is required to view it. Open it in any browser and it works.

2.1 USGS Mineral Commodity Summaries (MCS)

The USGS publishes the Mineral Commodity Summaries annually, covering ~90 mineral commodities. Each commodity chapter reports: US mine production, imports for consumption, exports, apparent consumption (production + imports − exports), and average price. The MCS is the authoritative US government source for non-fuel mineral statistics.

  • URL: https://pubs.usgs.gov/periodicals/mcs2024/ (replace year)
  • Format: PDF (annual), one PDF per commodity chapter
  • Cadence: Published each January for the prior calendar year (with preliminary estimates)
  • Access: Free, public domain (US government work)

2.2 AIST Steel Mill Directory

The Association for Iron & Steel Technology (AIST) publishes a directory of steel production facilities in North America. It includes plant type (blast furnace, EAF minimill, rolling, finishing), annual capacity, and geographic coordinates. The blast furnace facilities in this directory are the primary source of granulated slag — a co-product of iron reduction that, when quenched and ground, becomes GGBFS.

  • Format: GeoJSON (converted from AIST directory, 68 facilities, 56 KB)
  • Included file: data/facilities_steel_mill.geojson
  • Filter applied: Only facilities with "blast_furnace" in the facility_types property are used — 8 integrated BF-BOF mills that produce pig iron and, as a by-product, slag

3. Build Pipeline & Tools

The entire report — all charts, the map, all statistics — is generated by a single Python script: build_report.py. The build takes under 2 seconds and produces a fully self-contained HTML file requiring no web server.

3.1 Python Dependencies

duckdb (≥ 0.9.0)

In-process analytical database. Reads usgs_minerals.duckdb directly — no server process, connects to the file like SQLite. Used for all USGS MCS queries.

json (stdlib)

Parses facilities_steel_mill.geojson and serializes Python data structures into JavaScript-compatible inline constants embedded in the HTML.

statistics (stdlib)

Provides statistics.median() used to collapse multiple USGS data points for the same commodity/year into a single stable value (see §5 for deduplication logic).

pathlib (stdlib)

Resolves all file paths relative to the script location (Path(__file__).resolve().parent), making the repo work from any directory on any OS.

3.2 Frontend Libraries (CDN — no installation required)

Chart.js 4.4.0

Renders all 15 charts (bar, line, doughnut, pie) directly on HTML5 <canvas> elements. No build step. CDN: cdn.jsdelivr.net/npm/chart.js

Leaflet.js 1.9.4

Renders the interactive facility map. Handles tile loading, marker placement, layer controls, and click popups. CDN: unpkg.com/leaflet

CartoDB DarkMatter / Light tiles

Dark and light basemap tile sets, switched automatically when the user toggles the report theme. No API key required. Served from: basemaps.cartocdn.com

Space Grotesk (Google Fonts)

Geometric sans-serif typeface used for all text: body, labels, chart annotations, map popups. Weights: 300, 400, 500, 600, 700. CDN: fonts.googleapis.com

3.3 Build Steps

Step 1 — Load USGS data
      Open DuckDB connection to data/usgs_minerals.duckdb (read-only)
      Query mcs_salient_stats for each commodity/metric/year range
      Aggregate to single median value per year (see §5)
      Result: Python dicts ready for json.dumps()

Step 2 — Load AIST mill locations
      Parse data/facilities_steel_mill.geojson with json.load()
      Filter: country = "US", facility_types contains "blast_furnace"
      Extract: name, owner, city_state, lat, lon
      Result: list of 8 mill tuples

Step 3 — Build HTML string
      Inject all data as JavaScript const declarations (inline)
      Embed Chart.js datasets using the live data values
      Embed Leaflet map initialization code with all 4 layers
      Write single output file to output/

Output: ~56 KB standalone HTML (all data embedded as JS literals)
Why embed data as JavaScript constants? The HTML file is designed to be opened directly from disk (no web server). JavaScript fetch() calls are blocked by browsers for local files due to CORS policy. By embedding data directly as const declarations in an inline <script> block, the report works identically whether opened from disk, served via HTTP, or hosted on GitHub Pages.

4.4 DuckDB Table Schema

The mcs_salient_stats table is the core of the USGS database. Each row is one reported value from one MCS PDF:

mcs_salient_stats
  commodity     TEXT    -- e.g. 'cement', 'pumice', 'iron_steel_slag'
  year          INT     -- reporting year (2000–2024)
  metric        TEXT    -- e.g. 'production_us', 'imports_for_consumption', 'price_avg'
  value         FLOAT   -- the reported number (units vary by commodity)
  unit          TEXT    -- e.g. 'thousand metric tons', '$/metric ton'
  source_file   TEXT    -- which PDF this came from

Total: 32,344 rows across 37 commodities

5. USGS Data Methodology

5.1 Database Construction

The usgs_minerals.duckdb database (7.5 MB, included in this repo) was built by a scraper-parser pipeline that:

  1. Downloaded all available MCS PDF files from USGS servers (37 commodities, 4,238 files, ~1.86 GB of PDFs)
  2. Parsed each PDF using text-block extraction to locate salient statistics tables
  3. Extracted year, metric name, value, and unit for each row
  4. Loaded results into mcs_salient_stats (32,344 records)

The raw PDFs are not included — only the extracted database (7.5 MB). This keeps the repo small while retaining all queryable data.

5.2 Multi-Record Deduplication

A key challenge: the USGS publishes multiple PDF editions covering overlapping years (preliminary estimates get revised in subsequent publications). This means any given commodity/year combination may appear 3–15 times with slightly different values.

Resolution: For each commodity/metric/year, the build script filters to a plausible physical value range (e.g. cement production must be between 75,000–100,000 thousand metric tons), then takes the statistical median of all qualifying values. This produces a stable, revision-resistant estimate.

Example query for cement production:

SELECT year, value
FROM mcs_salient_stats
WHERE commodity = 'cement'
  AND metric = 'production_us'
  AND year BETWEEN 2020 AND 2024
  AND value BETWEEN 75000 AND 100000   -- plausibility filter (thousand MT)
ORDER BY year

-- Multiple values per year from different PDF editions:
-- 2022: [92000, 91500, 92300] from 3 separate PDFs
-- statistics.median() => 92000 => displayed as 92.0 MMT

5.4 Unit Normalization

USGS MCS values are typically reported in one of three scales:

  • Thousand metric tons — most production figures (cement, slag, pumice)
  • Metric tons — some price and smaller commodity figures
  • Dollars per metric ton — price series

The build script divides thousand MT values by 1,000 before displaying, showing all production in million metric tons (MMT) for consistency. Prices are shown as-is in USD/MT.

5.5 Cement-Specific Notes

  • USGS reports Portland and masonry cement combined — Portland is ~95% of that total
  • Apparent consumption = production + imports − exports (modeled as production × 1.22 for chart display)
  • Price data reflects the average value at the plant gate, not the delivered price to the customer

6. Facility & Location Data

6.1 Cement Plants (50 facilities — curated)

US cement plant locations and clinker capacities are sourced from PCA member data, USGS Cement commodity reports (which list major producers by state), and cross-referenced against SEC filings from major producers (Lehigh/Heidelberg, CEMEX, Martin Marietta, Summit Materials, Ash Grove/CRH). Coordinates are centroid estimates for the plant county.

Each plant record contains: [name, owner, state, capacity_MT, lat, lon]

Note: Cement plant data does not update automatically. The PCA does not publish a free machine-readable facility database. Capacities shown reflect conditions circa 2023–2024. Verify against the PCA Market Intelligence Report for precision work.

6.2 Import Ports (24 facilities — curated)

US cement-capable import terminals identified from:

  • USGS annual cement commodity reports (list importing states and ports)
  • USACE waterborne commerce statistics (cement/clinker tonnage by port)
  • Panjiva/S&P Global trade data (vessel-level clinker import records)

Each port record contains: [name, volume_MT, volume_label, shipments, lat, lon]

Port volumes indicate relative import activity, not exact annual tonnage.

6.3 Blast Furnace Mills (8 facilities — live from AIST GeoJSON)

These are the only US facilities currently producing molten slag — a by-product of reducing iron ore to pig iron in a blast furnace. When quenched with water, this slag granulates and can be ground to GGBFS. The 8 mills are queried live from data/facilities_steel_mill.geojson on every build.

Each mill record (after filtering): [name, owner, city_state, lat, lon]

Active US BF-BOF integrated mills as of 2024:

  • US Steel — Gary Works (IN), Granite City Works (IL), Great Lakes Works (MI)
  • Cleveland-Cliffs — Burns Harbor (IN), Indiana Harbor East (IN), Cleveland Works (OH), Middletown Works (OH), Dearborn Works (MI)

6.4 Natural Pozzolan Deposits (8 deposits — curated)

Deposit locations derived from USGS Mineral Resources Data System (MRDS) and USGS Professional Paper 1679 (Natural Pozzolans in the Western United States). All significant commercial-grade deposits are located in the western US due to Quaternary volcanic activity.

Each deposit record: [name, location, type, lat, lon]

7. Interactive Map — Technical Build

The infrastructure map (Section 6 of the report: "US Cementitious Materials Infrastructure") is built with Leaflet.js 1.9.4. It renders in the browser using HTML5 Canvas and SVG — no server processing required. The map is initialized from the same JavaScript constants that drive the charts, so all four layers update automatically on each build of build_report.py.

7.1 Map Initialization

L.map('mapContainer').setView([39.5, -98.5], 4)
-- Center: geographic center of the contiguous US
-- Zoom level 4: shows all 48 states at once
-- Div: <div id="mapContainer"> sized via CSS (height: 540px)

7.2 Basemap Tiles

The map uses CartoDB tile layers served via CDN. Two tile sets are loaded — the active one switches when the user toggles the report's dark/light mode:

Dark theme:  https://{s}.basemaps.cartocdn.com/dark_all/{z}/{x}/{y}{r}.png
Light theme: https://{s}.basemaps.cartocdn.com/light_all/{z}/{x}/{y}{r}.png

Subdomains: {s} cycles through a, b, c, d (load balancing)
Max zoom: 18 (street-level detail)
No API key required

The active tile layer is stored as window._tile. Theme toggle calls window._tile.setUrl() to swap tile sets without reloading the map.

7.3 Layer 1 — Cement Plants (green circles)

Cement Plants — 50 facilities
Source: PLANTS array (curated) • Marker type: L.circleMarker • Color: #4CAF50 (green)

Each cement plant is rendered as a filled circle whose radius scales with clinker capacity:

radius = Math.max(5, Math.min(14, capacity_MT * 4))
-- Minimum radius: 5px (plants under 1.25 MT/yr)
-- Maximum radius: 14px (plants over 3.5 MT/yr)
-- Linear scaling between: 1 MT = r5, 4 MT = r16 (capped at 14)

Clicking a circle opens a popup showing: plant name, owner, state, and capacity in MT/yr (highlighted in mint green).

The plant layer is added to plantsLyr (a Leaflet LayerGroup), which is added to the map by default and toggled via the layer control checkbox labelled "Cement Plants."

7.4 Layer 2 — Import Ports (red circles)

Import Ports — 24 ports
Source: PORTS array (curated) • Marker type: L.circleMarker • Color: #E53935 (red)

Each import port is a filled circle whose radius scales with import volume using a square-root transform to prevent the largest ports (Houston: 5.7M MT) from overwhelming smaller ones:

radius = Math.max(6, Math.min(30, Math.sqrt(volume_MT / 8000)))
-- Houston, TX: sqrt(5717149/8000) = 26.7 => r27
-- NY/Newark:   sqrt(856976/8000)  = 10.3 => r10
-- Mobile, AL:  sqrt(30016/8000)   = 1.9  => r6 (minimum)
-- Square root compresses the range from 190x to ~4.5x

Popup shows: port name, formatted volume (e.g. "5.72M MT"), and shipment count.

7.5 Layer 3 — Blast Furnace Mills (blue diamonds)

BF Steel Mills — 8 facilities (live from AIST GeoJSON)
Source: BF_MILLS array (live from facilities_steel_mill.geojson) • Marker type: L.marker with L.divIcon

BF mills use a rotated square (diamond) custom icon to visually distinguish them from the circular plant and port markers. The icon is a pure-CSS HTML element rendered via L.divIcon — no image file needed:

7.7 Layer Control

The custom layer control panel (top-right corner of the map) is built with L.control() and a manually constructed HTML string. It is not the built-in L.control.layers() — this gives full control over styling to match the report's dark theme:

-- Each checkbox wires to an event listener:
document.getElementById('tP').onchange = e =>
    e.target.checked ? map.addLayer(plantsLyr) : map.removeLayer(plantsLyr);
-- Same pattern for tR (ports), tB (BF mills), tZ (pozzolans)

L.DomEvent.disableClickPropagation(div) prevents map click events from firing when the user clicks inside the layer control panel.

7.8 Dark/Light Mode Tile Switching

When the user clicks the theme toggle button, two things happen simultaneously:

  1. CSS class light-mode is toggled on document.body — CSS variables swap all colors
  2. window.updateMapTiles() is called — swaps the tile layer URL between CartoDB Dark and Light

Chart colors are also updated via window.updateChartTheme(), which modifies Chart.defaults for grid lines, tick labels, and legend text.

9. Commodity Coverage Detail

9.1 Portland Cement

Cement is manufactured by heating limestone (CaCO₃) and clay minerals to ~1450°C in a rotary kiln, producing clinker. Clinker is then ground with ~5% gypsum to produce Portland cement. The US has ~99 plants producing ~88–93 MMT/year, meeting ~80% of domestic demand; the remainder is imported primarily from Canada, Turkey, Vietnam, Greece, and South Korea.

9.2 Iron & Steel Slag / GGBFS

Blast furnace slag is produced at approximately 250–350 kg per tonne of pig iron. With ~25 Mt of pig iron produced annually at US integrated mills, approximately 6–9 Mt of slag is generated. Around 70% is granulated (water-quenched → GGBFS feedstock); the remainder is air-cooled and sold as aggregate or road base. GGBFS reduces cement content by 30–70% in concrete mixes, lowering embodied carbon by a similar proportion.

9.3 Fly Ash

Fly ash is captured from the flue gas of coal-fired power plants. US fly ash supply has declined ~40% since 2010 as coal plant retirements accelerate — from ~70 Mt available annually to ~45 Mt. Not all captured fly ash meets ASTM C618 specifications; usable supply is approximately 30–35 Mt/year, of which ~18–20 Mt is consumed as a cement replacement. Supply constraints are the primary growth challenge for fly ash-based SCMs.

9.4 Natural Pozzolans

Natural pozzolans include pumice, volcanic ash, calcined clay (metakaolin), calcined shale, diatomite, and perlite. US production is modest (~0.5–1.0 Mt/year total for SCM applications) but growing as fly ash supply tightens. Key producers: Hess Pumice (ID), CR Minerals (NM), Burgess Pigment (metakaolin, GA).

10. Limitations & Caveats

  • USGS MCS preliminary data: The most recent year (e.g., 2024) reflects preliminary estimates that will be revised. Treat current-year figures as ±5% estimates.
  • Fly ash — no live data: The USGS does not publish a dedicated fly ash MCS chapter. Fly ash statistics are based on ACAA survey data and EPA RCRA reporting. These are curated estimates, not live DB queries.
  • Cement plant capacities: Clinker capacities reflect publicly reported figures, not PCA survey data (proprietary). Utilization rates vary significantly by plant and year.
  • GGBFS market share: GGBFS consumption is modeled from slag production (USGS) minus air-cooled uses, with a yield factor of ~0.95. No government agency tracks GGBFS consumption directly.
  • Map coordinates: Cement plant coordinates are county-centroid approximations in some cases; not suitable for precision logistics routing without verification.
  • AIST directory vintage: The GeoJSON is from the 2021 AIST edition. Mill ownership has changed (e.g., Cleveland-Cliffs acquisitions) and some mills have been idled or restarted since then. The BF mill list is directionally correct but may not reflect the exact 2025 status of all facilities.

11. Setup & Reproduction

11.1 Requirements

  • Python 3.11 or higher
  • duckdb Python package (≥ 0.9.0) — pip install duckdb
  • Internet connection when viewing the report (map tiles and CDN assets)
  • No internet connection required to build the report (all data is local)

11.2 Installation

git clone https://github.com/theshipsagent/construction-materials-intel
cd construction-materials-intel
pip install -r requirements.txt

11.3 Building the Report

python3 build_report.py
-- Output: output/construction_materials_intelligence_report.html (~56 KB)
-- Build time: under 2 seconds

Open the output file in any modern browser (Chrome, Firefox, Safari, Edge). For best results use Chrome or Firefox — Safari has stricter local file restrictions on some systems.

11.5 Refreshing USGS Data

The data/usgs_minerals.duckdb database is a snapshot. To refresh it with the latest USGS publications, the full USGS Intelligence pipeline (not included in this repo) must be re-run. Contact the repository maintainer for an updated database snapshot, or to request refresh procedures if you need the latest annual data.
